个人管理

发表于 2020-06-12 更新于 2020-07-15 分类于 个人管理

这个六月主要需要思考与整理清楚以下几件事：
※ 在哪些方面做好个人管理
※ 研究和迭代更先进的个人管理模式
※ 如何采用全线上模式的个人管理
※ 制定好更换新的工作生活环境后的作息和需求
※ 如何在生活中增加了一个人以后不打乱整体的工作节奏

个人管理

先做一些简单的整理

问题清单

以前的管理模式下存在以下问题

  • 每日待办反复积压，琐事拖延
  • 整体工作目标性不强
  • 未能实现专注力训练，一旦放松很长时间回复不了状态
  • 事务优先级未能确定，杂乱的顺序安排和没有全局的预估容易造成拖延
  • 单人生活的灵活性在增加了一个人以后节奏散乱，很长时间没能及时调整
  • 个人知识体系未能有效管理，懂的多忘的多，无法及时回顾

管理系统实现

六月想要实现纯线上的管理模式

资源库（tms_resource）

书本或者课程视频资源

字段 类型 描述 备注
id int(10) 主键
title varchar(100) 名称
description varchar(255) 描述
type tinyint 资源分类
category tinyint 媒体类型
status tinyint 完成状态
essence tinyint 精华资源
grade tinyint 资源评分
summary tinyint
create_time datetime 创建时间

备忘录（tms_memo）

随时记录的想法、待处理事件、备忘等。草稿只停供标题和详情，可以通过草稿生成目标、任务、备忘录。处理时间在一个时间单元以下或是暂时未规划时间，指定在特定时间要做的事情。

字段 类型 描述 备注
id int(10) 主键
title varchar(100) 标题
description varchar(255) 描述
status tinyint 状态
pop tinyint 菜单提示
create_time datetime 创建时间

目标库（tms_target）

在一定时间段内想要完成的事情、实现的目的、达到的标准，这个时间段可能是月、季、半年、年、长期

字段 类型 描述 备注
id int(10) 主键
title varchar(100) 标题
description varchar(255) 描述
type tinyint 类型
status tinyint 状态
period tinyint 周期
parent_id int(10) 父目标
parent_title varchar(100) 父标题
deadline tinyint 截止日期
create_time datetime 创建时间

例常库（tms_routine）

两周以及以下范围内的循环事件，可以指定每日、每周几、单双周几进行。
起床
三餐
睡觉
锻炼
计划
追踪
总结

字段 类型 描述 备注
id int(10) 主键
title varchar(100) 标题
description varchar(255) 描述
status tinyint 生效状态
period tinyint 周期
start_time tinyint 计划起始时间
planned_unit tinyint 计划消耗时间单元
create_time datetime 创建时间

事件清单（tms_event）

一次性的事项，处理耗时在一个时间单元以上，可以由例常、目标、备忘、草稿转换而来，也可以直接创建。例常演变的事项按照两周为周期拆解

字段 类型 描述 备注
id int(10) 主键
title varchar(100) 标题
description varchar(255) 描述
type tinyint(1) 类型
status tinyint(1) 状态
associated_id int(10) 关联ID
planned_unit varchar(10) 计划消耗时间单元
actual_unit varchar(10) 实际消耗时间单元
start_date varchar(10) 开始日期
end_date varchar(10) 截止日期
create_time datetime 创建时间

记录每天每个时间单元对应的任务。
