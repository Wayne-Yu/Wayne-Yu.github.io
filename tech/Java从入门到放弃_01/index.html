Java从入门到放弃（一）

发表于 2016-12-01 更新于 2020-04-28 分类于 技术

从基础的java开始整理技术栈，本章为Java入门及基础概念。

Java入门

Java简介

Java是由Sun公司于1995年推出的Java程序设计语言和Java平台的总称
Java平台由Java虚拟机和Java应用编程接口构成
分为三个体系

  • JavaSE（Java2 Platform Standard Edition) Java平台标准版
  • JavaSE（Java2 Platform Enterprise Edition) Java平台企业版
  • JavaME（Java2 Platform Micro Edition) Java平台标准版

Java特点

  • 简单 语法与C、C++接近，丢弃了操作符重载、多继承等，不再使用指针，提供垃圾收集机制
  • 面向对象 支持类之间的单继承和接口之间的多继承
  • 分布式 网络编程接口（java.net） 远程方法激活RMI
  • 健壮 强类型机制、异常处理、废料的自动收集 安全检查机制
  • 安全 ClassLoader类 字节码检查和安全管理机制
  • 可移植 Java的运行环境由ANSI C实现，可在多平台上运行
  • 解释型 Java程序被编译为字节码格式运行
  • 高性能 JIT（Just-In-Time）编译器技术
  • 多线程 支持多线程
  • 动态 Java程序中的类能够动态地被载入到运行环境中

名词解释

  • JDBC Java DateBase Connectivity 提供连接各种关系数据库的统一接口
  • EJB Enterprise JavaBeans 可以方便创建、部署和管理跨平台的基于组件的企业应用
  • Java RMI Java Remote Method Invocation 用来开发分布式Java应用程序
  • Java IDL Java Interface Definition Language
  • JNDI Java Naming and Directory Interface 提供Java平台到应用程序的统一无缝连接
  • JMAPI Java Management API 为异构网络上系统、网络和服务管理的开发提供一整套对象和方法
  • JMS Java Message Service 提供了企业消息服务
  • JTS Java Transaction Service 提供存取事务处理资源的开放标准
  • javaBean 一个开发的、标准的组件体系结构

编译和运行

javac用来编译Java程序，在destdir下生成.class文件。

通过java命名运行java程序

Java严格区分大小写，Java程序的关键字全部是小写

Java运行机制

编译型和解释型

  • 编译型 C、C++等
    运行效率高，难以移植到其他平台。
    编译型语言使用编译器针对特定OS将某种高级语言源码一次性Compile为可被该平台硬件执行的机器码，脱离开发独立运行。
    链接用来组装两个以上的目标代码模块生成最终的可执行程序，实现低层次代码复用。
  • 解释性 Ruby、Python等
    方便实现源程序级移植，牺牲程序执行效率为代价。
    解释性语言使用解释器对源程序逐行解释成特定平台的机器码并立即执行，不能脱离解释器独立运行。只需特定平台的解释器就可以跨平台运行。

Java运行机制

Java程序先经过编译步骤生成与平台无关的字节码（*.class文件），再通过Java虚拟机（JVM）来解释执行。不同平台的JVM不同，但提供了相同接口。

其他

源文件命名规则

Java程序源文件名必须与public类名相同，文件后缀名必须是”.java”，一个Java源文件只能定义一个public类。

main()方法

如果需要Java解释器直接运行一个Java类，该类须包含main()方法。必须用public和static修饰main()，必须用void声明其返回值，形参只能是一个字符串参数。

Java的垃圾回收机制

在Java中，但还没有对象引用指向原先分配给某个对象的内存时，该内存便成为垃圾，JVM的一个系统级线程会自动释放该内存块。

JDK和JRE

JDK（Java Development Kit）是Java核心，包含Java运行环境、Java工具和Java基础类库。

面向对象

类之间的关系

  • 继承inheritance 一个类从其父类派生，继承父类的属性和方法。基于类的继承叫一般化（generalization），基于接口的继承叫实现（realization）。
  • 关联association 类之间的关联大多数用来表示变量实例持有对其他对象的引用，半永久的无包含关系。
  • 依赖dependency 不同类实例之间的暂时相互关系。
  • 聚合aggregation 关联的特殊形式，整体和部分的关系，部分可以作为其他整体的部分，无生命期依赖。
  • 组合composition 聚合的特殊形式，部分只能作为唯一一个整体的部分，部分的生命周期依赖于整体的生命周期。
